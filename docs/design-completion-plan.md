## A. 즉시 디자인 보강 (코드 있는 페이지)

### A1. `/login` — 로그인

**현재 상태**: shadcn `<Card>` + email/password Input + Google 버튼. 화면 가운데 정렬, max-w-md. 기능 작동.

**진단**:
- 카드 한 개만 있어 시각적 무게감 부족
- Hero 영역 / 브랜드 정체성 부재 (로그인 페이지가 외관상 "Next.js 기본 예제"처럼 보임)
- 로딩 상태 spinner 빈약 ("처리 중..." 텍스트만)

**디자인 목표**:
- Apple ID 로그인 / Linear / Notion 로그인 스타일 — 화면 좌측 브랜드 + 우측 폼 split layout (데스크탑) / 모바일 단일 컬럼
- Pretendard 큰 타이포로 "다시 만나서 반갑습니다" 같은 환영 카피
- shadcn `Card` 제거 → 토큰 직접 활용한 그라데이션 없는 깔끔한 폼

**레이아웃** (데스크탑 ≥ md):
```
┌──────────────────────┬────────────────────────┐
│                      │  ← 상단 워드마크 (작게)│
│  [좌측 브랜드 영역]   │                        │
│                      │  text-display          │
│  Stethoscope 아이콘  │  "다시 만나서 반갑습니다"│
│  큰 톤다운 일러스트  │                        │
│  또는 단일 컬러 패널 │  Email Input           │
│  bg-secondary        │  Password Input        │
│  (50% width)         │  [이메일로 로그인]      │
│                      │  ─── 또는 ───          │
│  하단 안내 1줄       │  [Google로 계속]       │
│                      │                        │
│                      │  계정이 없으신가요? 가입│
└──────────────────────┴────────────────────────┘
```
모바일 (< md): 브랜드 영역 숨김, 폼만 풀스크린 가운데.

**핵심 컴포넌트**:
- 좌측 패널: `bg-[var(--color-bg-secondary)]` + 중앙 큰 `Stethoscope` 아이콘 (Lucide w-24 h-24, 색 `text-accent`) + "MedPlace" 워드마크 + 한 줄 카피
- 우측 폼: `h-12` 둥근 input + `rounded-full` Apple pill 버튼 + 상단 워드마크 (`Link href="/"` 로고 회귀)
- 로딩: 버튼 내부 `Loader2` spinner (Lucide) + label 변경

**인터랙션**:
- focus ring: `focus-visible:ring-2 focus-visible:ring-[var(--color-accent)]/30`
- 에러: input 좌측 빨간 border + 메시지 아이콘 `AlertCircle`
- 자동 포커스: 페이지 마운트 시 email input

**작업 단위**: ~2시간 (split layout + 토큰 통일 + spinner)

### A2. `/register` — 회원가입

**현재 상태**: shadcn `<Card>` + role 라디오 카드 2개 (병원/공급업체) + 이름·이메일·비밀번호 폼 + Google 버튼.

**진단**:
- role 라디오 카드는 좋은 패턴이나 시각적 비중이 작음 (3행 텍스트)
- 4개 필드 + 라디오가 한 화면에 압축되어 답답함
- 가입 후 onboarding 진입 흐름이 어디로 가는지 사용자에게 시각적 단서 없음

**디자인 목표**:
- **2-step 진행** UX 도입 — Step 1: 역할 선택 (큰 카드 2개) → Step 2: 폼 입력
- 역할별로 카드 배경 색을 살짝 다르게 (병원=`accent-light`, 공급업체=`bg-tertiary`) + 카드 안에 페르소나 일러스트(Lucide `Stethoscope` / `Building2` w-12 h-12)
- 폼 단계에서는 진입할 onboarding 경로 미리 안내 ("다음 단계: 사업자 인증 4단계" 등)

**레이아웃**:
```
Step 1 — 역할 선택:
┌──────────────┐  ┌──────────────┐
│   [아이콘]    │  │   [아이콘]    │
│   병원·의원   │  │   공급업체    │
│   설명 2줄    │  │   설명 2줄    │
│   [이 역할로] │  │   [이 역할로] │
└──────────────┘  └──────────────┘

Step 2 — 폼:
   ← 역할 변경
   "병원 계정 만들기"  (또는 "공급업체 계정 만들기")
   
   이름  [______________]
   이메일 [______________]
   비밀번호 [_____________]
   
   [무료로 가입하기]
   ─── 또는 ───
   [Google로 가입]
   
   가입 시 다음 단계로 진행됩니다:
   1. 이메일 인증 (자동)
   2. 사업자 등록증 업로드
   3. 사업자 진위 확인
   4. 가입 완료
```

**핵심 컴포넌트**:
- Step indicator: 상단 진행 점 2개 (active = `bg-accent`, inactive = `bg-border-light`)
- 큰 role 카드: hover 시 shadow-md + active:scale-[0.98]
- "← 역할 변경" 텍스트 버튼 (Lucide `ArrowLeft`)

**인터랙션**:
- 역할 선택 후 부드러운 step 전환 (CSS opacity + max-height transition, 200ms)
- 폼 제출 시 버튼이 spinner로 변환 + "계정 만드는 중..." (한글)

**작업 단위**: ~3시간 (2-step refactor + 카드 디자인 + 안내 텍스트)

### A3. `/onboarding/buyer` — 4-step wizard

**현재 상태**: Stepper 4단계 + 사업자등록증 업로드 / 병원 정보 / 검증 / 완료. Card 안에 모든 UI 압축.

**진단**:
- Stepper가 단순한 4개 막대 (현재 단계 라벨만 표시)
- 각 step 의 Card가 같은 크기로 답답하게 보임
- Step 3 "검증 중" 화면은 단순 `animate-pulse` 점 1개 — 시각적 무게감 부족
- 완료 화면은 평범한 안내 카드

**디자인 목표**:
- Apple 가입 / Stripe Connect 스타일 진행 시퀀스 — 좌측 Stepper (데스크탑) + 우측 콘텐츠
- Step 3 "검증 중" — 3개 사이드 진행 표시 (`OCR 분석 → 국세청 조회 → 등록 완료`) — 실시간 표시
- Step 4 "완료" — `CheckCircle2` 큰 아이콘 + 입금된 등록 정보 요약 표 + 다음 액션 큰 CTA

**레이아웃** (데스크탑):
```
┌──────────────────┬───────────────────────────────┐
│ 진행 단계         │  Step N — [현재 단계 타이틀]   │
│                  │                                │
│ ✓ 1. 사업자등록증 │  콘텐츠 영역                   │
│ ● 2. 병원 정보    │  (입력 폼 / 검증 진행 / 완료)  │
│ ○ 3. 검증         │                                │
│ ○ 4. 완료         │                                │
│                  │                                │
│ "처음 5분이면     │  [뒤로]      [다음]            │
│  충분합니다"     │                                │
└──────────────────┴───────────────────────────────┘
```

모바일: 상단 가로 stepper + 단일 컬럼.

**Step 3 검증 화면 (특별 디자인)**:
```
   [Pulse 큰 원형 spinner — accent 색, 64x64]
   
   사업자 정보를 확인하고 있습니다
   
   ✓ 사업자등록증 인식 완료
   ⟳ 국세청 진위 확인 중
   ○ 등록 처리 대기
```

**핵심 컴포넌트**:
- 좌측 Stepper: 각 step 동그라미 아이콘 + 텍스트 + 상태별 색 (완료=`accent`, 현재=`accent` 채움, 대기=`border` 빈)
- 진행 상태 텍스트는 `text-h3` 18px 정도

**인터랙션**:
- step 이동 시 200ms slide transition (`translateX(20px) → 0` + opacity)
- "검증 중" 상태에서 각 sub-step 완료 시 부드러운 ✓ 아이콘 fade-in
- 파일 업로드: drag-and-drop 영역 (점선 테두리 + Upload 아이콘) + 파일 선택 후 미리보기

**작업 단위**: ~4시간 (Stepper 컴포넌트 분리 + Step 3 검증 단계 시각화 + Step 4 완료 화면)

### A6. `/admin/vendors` — 입점 심사 큐

**현재 상태**: 상단 status 필터 탭 4개 + 테이블 (회사명, 구분, 사업자번호, 대표자, 신청일, 액션).

**진단**:
- 테이블 자체는 깔끔 (Stripe Dashboard 스타일에 근접)
- **검색·정렬·기간 필터 없음** — 입점 신청이 늘어나면 운영자 업무 불편
- 빈 상태(empty state) 디자인이 평범
- 통계 카드 (지금까지 승인 N건, 평균 처리 시간 X시간 등) 없음

**디자인 목표**:
- Linear / Stripe Dashboard 의 admin 뷰 패턴 — 상단 통계 카드 4개 + 필터 영역 + 테이블

## 작업 진행 방식 (Claude Code 제안)

각 페이지 보강은 다음 4단계 순환:
1. **현재 상태 스크린샷** (사용자가 dev 서버에서 캡처)
2. **Claude 가 diff 형태로 변경 사항 제시** (코드 작성 전 plan)
3. **사용자 승인 → Claude 가 코드 작성** (헌법 §9.3 절차)
4. **사용자가 다시 dev 서버에서 검증** + 다음 페이지로

**스타일 일관성을 위한 공통 컴포넌트 사전 추출** (한 번에 작성, 모든 페이지에서 재사용):
- `<PageHeader>` — `<p text-xs uppercase>` Phase 라벨 + `<h1>` 큰 타이틀 + 부제
- `<Section>` — `py-24 md:py-32` 큰 여백 섹션 컨테이너
- `<StatCard>` — A6 통계 카드 4개에서 재사용
- `<WizardStepper>` — A3/A4 좌측 stepper (수평/수직 모두 지원)
- `<EmptyState>` — A6 빈 상태 + 다른 admin 페이지에서 재사용
- `<TimelineList>` — A3 검증 단계 + A4/A5 진행 표시 공유

이 6개 공통 컴포넌트를 먼저 `src/components/shared/` 에 작성하면 A1~B3 작업 시간이 ~25% 절감됨.

   - 단, "운영자 효율"이 더 시급하면 A6/A7 먼저

3. **각 페이지 보강 후 사용자 승인 받고 다음 진행 vs A1~A7 일괄 진행 후 사용자 검증?**
   - 권장: 페이지별 승인 (헌법 §9.3 정신)
   - 대안: 일괄 진행 + 마지막 일괄 검증 (속도 ↑)

4. **이미지·일러스트 자산은 어떻게 할까?** (예: A1 좌측 패널 배경, B1 about Hero)
   - 옵션 A: 자산 없이 컬러·아이콘만 (현재 가능)
   - 옵션 B: AI 생성 일러스트 (Midjourney/DALL-E — 사용자 작업)
   - 옵션 C: Unsplash 무료 stock (라이선스 확인 필요)
   - 옵션 D: 추후 디자이너 외주
