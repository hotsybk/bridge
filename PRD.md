## 0. 사용자 정의

### 0.1 주요 사용자 (Persona)

| 페르소나 | 설명 | 핵심 Pain | 우리가 제공하는 가치 |
|---|---|---|---|
| **김원장 (의원)** | 1~30 병상, 원장이 직접 구매 결정 | 매번 다른 도매상 전화 주문, 가격 비교 불가, 정기 발주 까먹음 | 가격 비교 + 정기구독 자동발주 + 공동구매 절감 |
| **박팀장 (중소병원 원무팀)** | 30~100 병상, 다단계 결재 필요 | 견적 받기 번거로움, 세금계산서 수기 정리 | 결재 워크플로우 + 자동 세금계산서 + 지출 분석 |

| **운영자 (플랫폼)** | MedPlace 운영팀 | 입점 심사·분쟁 조정·정산 관리 | 통합 관리자 콘솔 |

## 2. 구매자(병원) 측 기능 명세

### 2.1 회원가입 & 사업자 인증 [P0]

**플로우**:
1. 이메일/비밀번호 또는 카카오 OAuth로 가입
2. 병원 정보 입력: 사업자등록번호, 요양기관번호, 병원명, 병원 유형(의원/중소병원/종합/상급종합)
3. 사업자등록증 이미지 업로드 → Naver Clova OCR → 사업자번호 자동 추출 → **국세청 휴폐업 조회 API**로 진위 확인
4. 검증 성공 시 즉시 가입 완료
5. 첫 로그인 시 결재 워크플로우 설정 (선택)

**필드**:
- 사업자등록번호: 10자리 (123-45-67890 형식)
- 요양기관번호: 8자리 (선택 — 종병급은 필수)
- 대표자명, 연락처, 주소
- 직원 초대(최대 5명, MVP 기준)

**예외 처리**:
- OCR 실패 시 → 수동 입력 + 운영자 검토 큐
- 휴폐업 상태 → 가입 거부 + 안내 메시지

### 2.2 홈 / 카테고리 / 검색 [P0]

**홈 페이지 섹션**:
- Hero: "병원 운영의 모든 것, 한 곳에서" (Apple 스타일 큰 타이포)
- 카테고리 타일 (대분류 6~8개)
- 진행 중인 공동구매 (3~6개 카드)
- 정기구독 추천 (로그인 시 — 사용자별 맞춤)
- 베스트 셀러 (카테고리별 Top 5)

**카테고리 트리** (3단계):
```
의료소모품
├── 일회용 의료용품 (장갑, 마스크, 가운)
├── 주사·수액 용품
├── 드레싱 (거즈, 반창고, 붕대)
├── 검사·진단 용품 (스왑, 키트)
└── 위생용품 (소독제, 손소독제)

의료기기
├── 진단기기 (혈압계, 청진기, 체온계)
├── 처치기구 (메스, 가위, 핀셋)
├── 모니터링 장비 (심전도, SpO2)
└── 영상·진단 장비

기타
├── 한방용품
├── 치과용품
├── 응급·구급
└── 사무·청구 용품
```

**검색**:
- Meilisearch 기반, 한글 토크나이저
- 자동완성 (입력 2자부터)
- 필터: 카테고리, 등급(1~4), 인증서 유무, 가격, 공동구매 진행 중, 정기구독 가능, 즉시배송 가능
- 정렬: 인기순(기본), 낮은가격, 높은가격, 최신, 평점

### 2.3 상품 상세 [P0]

**구성**:
- 상품명, 브랜드, 공급업체명 (신뢰도 ★)
- 이미지 갤러리 (최대 10장)
- 가격 표시:
  - 단일가 또는 **수량별 가격 티어 표** (예: 1~9개 1,000원 / 10~99개 900원 / 100개+ 800원)
  - 진행 중 공동구매가 (있는 경우 강조 표시)
  - 정기구독 시 추가 할인율
- 핵심 정보:
  - UDI 코드
  - 식약처 허가/인증/신고 번호
  - 등급 (1~4)
  - 제조사·원산지
  - 인증서 PDF 다운로드 링크
  - 단위(EA/BOX/CASE), MOQ
- 재고 상태 (충분 / 한정 / 입고대기)
- 배송 정보 (예상 배송일, 배송비 정책)
- 구매 옵션 버튼:
  - "장바구니"
  - "바로 구매"
  - "정기구독으로 받기" (해당 상품만)
  - "공동구매 참여" (진행 중일 때)
  - "견적 요청" (5,000원 이상 고가품)
- 상품 설명, 사용법, 주의사항
- 리뷰 (병원 인증 사용자만 작성 가능)

### 2.4 장바구니 [P0]

**핵심 동작**:
- **벤더별 자동 그룹핑** — 같은 vendor 상품끼리 묶어서 표시
- 벤더별 소계, 배송비, MOQ 충족 여부 표시
- MOQ 미달 시 경고 메시지 + 추가 수량 제안
- 수량 변경 시 가격 티어 재계산 (실시간)
- 쿠폰 적용 (벤더별 또는 전체)
- "공동구매로 전환" 버튼 (해당 상품)

**Redis 저장 구조**: `cart:{userId}` → 30일 TTL

### 2.5 주문서 & 결제 [P0]

**주문서 페이지 구성**:
1. 배송지 (병원 주소 기본, 분원 선택 가능)
2. 벤더별 주문 요약 (SubOrder 분할 미리보기)
3. 결제 수단:
   - 신용카드 (즉시 결제)
   - 계좌이체 (즉시 결제)
   - **세금계산서 후불 결제** (Net30 — 신용도 기반 활성화, 초기에는 운영자 수동 승인)
4. 세금계산서 발행 정보 (사업자 정보 기본값 + 이메일)
5. **결재 워크플로우** — 일정 금액 이상이면 자동으로 결재요청 상태로 전환
6. 주문자 메모 (선택)

**결제 처리** (PortOne V2):
- 1 paymentIntent로 전체 결제 (총액)
- 결제 성공 시 `Order` 1건 + `SubOrder` N건(vendor 수) 생성
- 각 SubOrder는 독립적인 상태 전환 가능

**결재 워크플로우** (Phase 2 후반):
- 병원이 설정한 결재 한도 초과 시 → `Order.status = PENDING_APPROVAL`
- 결재자(BUYER_OWNER)에게 알림톡 + 이메일
- 결재 승인 시 → 결제 진행 또는 자동 결제(미리 카드 등록한 경우)

### 2.6 주문 이력 [P0]

**리스트 페이지**:
- 기간 필터 (1주/1개월/3개월/직접)
- 상태 필터 (전체/처리중/배송중/완료/취소)
- 벤더 필터
- 통합 검색

**상세 페이지**:
- Order 상위 정보
- SubOrder별 카드 (벤더별 상태 추적)
- 배송 추적 링크 (택배사 API 연동)
- 세금계산서 다운로드/재발행
- 환불/반품 요청
- 재주문 버튼

### 2.7 정기구독 관리 [P1] — Phase 3

**구독 생성 플로우**:
1. 상품 상세 또는 주문 완료 후 "정기구독으로 받기" 클릭
2. 주기 선택:
   - 매주
   - 격주
   - 매월
   - 사용자 정의 (예: 매월 5일, 매월 마지막 영업일)
3. 수량 설정 (수량별 가격 티어 즉시 반영)
4. 자동 결제 수단 등록 (PortOne 빌링키) 또는 외상거래 모드
5. 자동 승인 여부 (BUYER_STAFF가 만든 구독은 OWNER 승인 필요)

**구독 관리 페이지**:
- 활성 구독 카드 리스트 (다음 발주일 D-day)
- 캘린더 뷰 (월별 자동 발주 일정)
- 일시정지 / 1회 스킵 / 수량 변경 / 취소
- 발주 이력

**자동 발주 동작** (Cron 매일 03:00 KST):
1. `Subscription.nextRunAt <= now()` 쿼리
2. 재고 확인 → 부족 시 알림톡 + 대체 추천
3. 가격 변동 확인 → 5% 이상 변동 시 사용자 승인 요청
4. 자동 주문 생성 (또는 승인 큐 등록)
5. `nextRunAt += cadence` 갱신

### 2.8 공동구매 [P1] — Phase 4

**리스트 페이지**:
- 진행 중 (마감 임박순)
- 참여 가능
- 종료 (참고용)

**상품 카드**:
- 진행률 바 (예: 67% — 67/100 BOX 모임)
- 카운트다운 (D-3 14:30:00)
- 가격 티어 표시 (현재 가격 / 다음 단계 도달 시 가격)
- 참여 병원 수 (개수만, 익명)

**참여 플로우**:
1. 수량 선택
2. PortOne **사전승인(pre-auth)** — 카드 hold만, capture는 마감 후
3. 마감 시점:
   - **목표 도달**: 일괄 capture → SubOrder 생성 → 배송
   - **미달**: 일괄 void → 알림톡 "이번 공동구매는 미달로 종료되었습니다"

**Race Condition 처리**:
- Redis `INCR groupbuy:{id}:qty`로 원자적 카운트
- DB는 Outbox 패턴으로 비동기 동기화
- 마감 시점 Redis 값 기준으로 일괄 처리

### 2.9 견적요청 (RFQ) [P1] — Phase 5

**용도**: 카탈로그에 없는 상품 또는 대량/맞춤 구매

**작성 플로우**:
1. 제목 (예: "심전도 모니터 10대 견적")
2. 카테고리·하위카테고리 선택
3. 스펙 (자유 입력 + 첨부 파일)
4. 수량
5. 희망 납기일
6. 견적 마감일 (기본 7일)
7. 공개 여부 (전체 vendor 또는 특정 vendor 지명)

**견적 응답 (vendor 측)**:
- 단가, 총액, 납기일, 유효기간, 비고

**병원 측 비교**:
- 표 형태로 N개 견적 한눈에 비교
- 채택 시 자동으로 주문 전환

### 2.10 결재 워크플로우 [P1]

**설정**:
- 결재 한도 (예: 100만 원 초과 시 결재)
- 결재 라인 (1단계: STAFF→OWNER, 2단계: STAFF→실장→OWNER 등)
- 자동 결제 카드 등록 (결재 후 자동 결제용)

**진행**:
- BUYER_STAFF 주문 → 결재 한도 초과 시 PENDING_APPROVAL
- 결재자에게 알림톡 + 이메일
- 모바일에서도 1탭 승인 가능
- 거부 시 사유 입력 필수

### 2.11 지출 분석 [P2]

- 월별/카테고리별 지출 추이 차트
- 절감액 표시 (정가 vs 실구매가, 공동구매 시 추가 절감)
- 정기구독 발주 누적 통계
- 세금계산서 모음 (월별/연별)
- CSV 다운로드

## 3. 판매자(공급업체) 측 — "셀러센터" 기능 명세

### 3.1 입점 가입 & 심사 [P0]

**가입 플로우**:
1. 이메일/카카오 로그인
2. 회사 정보 입력:
   - 사업자등록번호
   - 회사명, 대표자, 연락처, 주소
   - **의료기기 판매업 신고증 번호** (의료기기법 제17조)
   - 영업 카테고리 (의료기기 / 의료소모품 / 한방 / 치과 등)
3. **필수 서류 업로드**:
   - 사업자등록증 (Clova OCR)
   - 의료기기 판매업 신고증 (OCR로 신고번호 추출)
   - 통장 사본 (정산 계좌)
4. 운영자 심사 대기 (24~72시간)
5. 승인 시 셀러센터 접근 가능

**예외 처리**:
- 제조·수입업자가 직접 판매하는 경우 판매업 신고 면제 (의료기기법 제17조) → 별도 체크박스 + 제조·수입업 허가증 업로드

### 3.2 상품 등록 [P0]

**등록 폼**:
- **기본 정보**: 상품명, 영문명, 카테고리, 브랜드
- **이미지**: 최대 10장 (대표 1장 필수, R2 업로드)
- **인증 정보** (필수):
  - UDI 코드 (GS1 형식 검증)
  - 식약처 허가/인증/신고 번호
  - 등급 (1~4)
  - 인증서 PDF 업로드
- **가격 정책**:
  - 단일가 또는 수량 티어 (최대 5단계)
  - MOQ
  - 단위 (EA/BOX/CASE/SET)
- **재고**: 현재 수량 또는 무제한(주문 후 발주)
- **배송**: 자체 배송 / 택배 / 직배송, 기본 배송비
- **추가 정보**: 상품 설명, 사용법, 주의사항, 제조사, 원산지, 유통기한 정책

**CSV 일괄 등록**: Phase 2.5 — 50개 이상 보유 vendor용

### 3.3 주문 접수 & 처리 [P0]

**대시보드**:
- 신규 주문 카운트 (실시간 Firestore onSnapshot)
- 처리 대기 / 출고 준비 / 배송 중 / 완료 탭

**SubOrder 처리 플로우**:
1. **신규 접수**: 알림톡 + 화면 토스트 → 클릭하여 상세
2. **출고 준비**: 상태 변경 (자동으로 buyer에게 알림톡)
3. **송장 입력**:
   - 택배사 선택
   - 운송장 번호
   - **LOT 번호 + 유통기한 입력 (UDI 자동보고용)**
4. **배송 완료**: buyer 확인 또는 자동 전환 (배송 후 3일)

### 3.4 정산 [P0]

**정산 페이지**:
- 정산 대기 잔액 (D+7 기본)
- **빠른정산 신청** (D+3, 매출액 90%, 수수료 일 0.012% 안내)
- 정산 이력 (월별, 다운로드 가능)
- **세금계산서 자동 발행** (PortOne 역발행) — 매월 1일 전월 분 일괄
- 수동 발행 옵션 (특정 주문 단건)

**계산 방식** (PortOne 공식 공식 차용):
```
정산금 = SubOrder 총액 
       - 결제수단 부가세 부담금
       - 중개수수료 (카테고리별 %)
       - 중개수수료 부가세
       - 추가수수료 (있는 경우)
       - 할인분담금 (쿠폰)
```

## 5. 비기능 요구사항

### 5.1 성능
- 페이지 로드: LCP < 2.5s (모바일 4G)
- 검색 응답: p95 < 200ms
- 결제 응답: p95 < 3s (PortOne 포함)
- 동시 접속: 10,000 (1년 목표)

### 5.2 가용성
- 운영 시간: 24/7
- 목표 가동률: 99.5% (Vercel + Firebase SLA 기준)
- 결제 장애 시 fallback: PortOne 멀티 채널 (토스/카카오/네이버 동시 활성)

### 5.3 보안
- 모든 통신 HTTPS
- 비밀번호: bcrypt 12 rounds
- 세션: HttpOnly, Secure, SameSite=Lax
- CSRF 방어 (next-firebase-auth-edge 서명 쿠키 + SameSite=Lax + httpOnly)
- Rate limiting (Upstash): 로그인 5회/분, API 100회/분
- Injection 방어: Firestore Security Rules + tRPC Zod 검증 기본 방어
- XSS: React 기본 + DOMPurify (사용자 HTML 입력 시)

### 5.4 컴플라이언스
- 개인정보보호법: 동의 페이지, 처리방침
- 전자상거래법: 통신판매업 신고번호 푸터
- 의료기기법: 판매업 신고증 검증 필수
- 전자세금계산서: 작성일 다음달 10일까지 국세청 전송
- UDI: 공급한 달 다음달 말일까지 식약처 보고

### 5.5 접근성
- WCAG 2.1 AA 기준
- 키보드 네비게이션 전체 지원
- 색상 외 상태 표시 (아이콘/텍스트 병행)
- 모바일 우선 (60%+ 트래픽 예상)
